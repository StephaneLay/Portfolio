BubbleDrop

We worked on this game for 6 months using the Godot Engine. I was involved in all aspects of development except the art. The most interesting work I did on this project includes:

  1. The grid system I designed that keeps track of each bubble in its coordinates and can process any desired effect on the structure upon collision.
  2. The A* pathfinding algorithm to drop bubbles that are no longer linked to the central node, then used by the scoring system , fully customizable with a mathematical expression.

This game will soon be available on Android. In the meantime, here's the French beta test if you like to give it a try

Level Design Plugin

This plugin I designed aims to simplify level design in every aspect. First, I created three custom docks on the right side of the interface, which are generated when using the TileMap node in Godot, allowing users to efficiently paint level tiles.

When a new level is created or an existing level is loaded, the save level section will require a level name, various threshold values used by the game's reward system, and a maximum number of attempts. The plugin will then calculate the scores for this level, extract all the coordinates from the tilemap corresponding to the appropriate colors, and efficiently save all this data in an optimized format.

The load level and manage data sections allow users to load, remove, or rename existing levels, each featuring an integrated custom search tool for easy navigation.
